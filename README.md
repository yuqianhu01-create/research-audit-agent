# Anti-Aging Research + Audit Agent Starter

This is a minimal demo agent for the anti-aging weekly briefing workflow.

It has two small agents:

1. Research Agent: searches for candidate records.
2. Audit Agent: checks link availability, date, duplicate status, and basic source validity.

The goal is not to replace the final enterprise workflow yet. The goal is to run a simple proof of concept, then use the result to design the company-level solution.

## Files

```text
instructions/
  general_instructions.md     # paste your full GENERATION_INSTRUCTIONS.md here
  briefing_rules.md           # short rules used directly by the demo agents
data/
  published_registry.json     # used articles, to prevent repetition
  candidates.json             # generated by Research Agent
  audit_report.json           # generated by Audit Agent
scripts/
  research_agent.py           # finds candidate academic articles
  audit_agent.py              # validates candidates
.github/workflows/
  weekly_agent.yml            # GitHub Actions schedule/manual run
```

## How to Run Locally

```bash
python scripts/research_agent.py --section academic --limit 10
python scripts/audit_agent.py --input data/candidates.json
```

## How to Run on GitHub

1. Upload all files in this folder to your GitHub repository.
2. Open the repository page.
3. Click `Actions`.
4. Click `Weekly Research Audit Agent`.
5. Click `Run workflow`.
6. After it finishes, download the artifact named `agent-output`.

## What This Demo Does

For the first version, the Research Agent only searches PubMed for the academic section. This is intentional: it makes the first test small, visible, and easy to debug.

After this works, expand sources:

- Crossref and journal pages for academic records.
- ClinicalTrials.gov for clinical and industry records.
- Company official pages and regulator websites for industry records.
- Product official pages and credible professional sources for AI application records.

## What This Demo Does Not Do Yet

- It does not write to Feishu Bitable.
- It does not generate the public website.
- It does not use OpenAI to write summaries yet.
- It does not perform deep semantic consistency checks.

Those belong to the enterprise version after the proof of concept is stable.
